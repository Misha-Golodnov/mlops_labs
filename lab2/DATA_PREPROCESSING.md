# Обработка данных - Wine Quality Dataset

## Описание

Этот документ описывает процесс обработки и подготовки данных для модели машинного обучения по качеству красного вина (Wine Quality Red Dataset из UCI Machine Learning Repository).

## Этапы обработки

### Этап 1: Загрузка данных
- **Источник**: https://archive.ics.uci.edu/ml/machine-learning-databases/wine-quality/winequality-red.csv
- **Размер**: 1599 строк, 12 колонок (11 признаков + 1 целевая переменная)
- **Формат**: CSV с разделителем `;`
- **Целевая переменная**: `quality` (оценка качества вина от 0 до 10)

### Этап 2: Обработка данных

#### 2.1 Обработка пропусков
- **Результат**: Пропусков не обнаружено в датасете

#### 2.2 Удаление выбросов
- **Метод**: Z-score с порогом 3.0 стандартных отклонений
- **Результат**: Удалено 10 выбросов, осталось 1589 строк

### Этап 3: Выделение важных признаков

Использован алгоритм Random Forest для определения важности каждого признака:

| Признак | Важность | % |
|---------|----------|-----|
| alcohol | 0.2782 | 27.8% |
| sulphates | 0.1396 | 14.0% |
| volatile acidity | 0.1113 | 11.1% |
| total sulfur dioxide | 0.0825 | 8.3% |
| chlorides | 0.0650 | 6.5% |
| pH | 0.0612 | 6.1% |
| residual sugar | 0.0554 | 5.5% |
| density | 0.0553 | 5.5% |
| citric acid | 0.0527 | 5.3% |
| fixed acidity | 0.0515 | 5.1% |
| free sulfur dioxide | 0.0474 | 4.7% |

**Вывод**: Все 11 признаков имеют значимую важность (> 1%). Наиболее важный признак - содержание алкоголя (27.8%).

### Этап 4: Формирование train/test выборок
- **Метод**: stratified train_test_split
- **Тестовая доля**: 20%
- **Random state**: 42 (воспроизводимость)
- **Стратификация**: По целевой переменной `quality`

Результаты:
- Training set: 1271 строка
- Test set: 318 строк

### Этап 5: Масштабирование признаков
- **Метод**: StandardScaler (z-score нормализация)
- **Формула**: $x_{scaled} = \frac{x - \mu}{\sigma}$
- **Параметры**: Scaler обучен на train, применен к test

**Результат**: Все признаки имеют среднее = 0 и стандартное отклонение = 1

### Этап 6: Отбор важных признаков
- **Выбрано признаков**: все 11 признаков (все имеют значимую важность)

## Выходные файлы

### Датасеты
- **X_train.csv** (1271 × 11): Признаки для обучения модели
  - Значения нормализованы (масштабированы)
  - Разделители: запятая
  
- **X_test.csv** (318 × 11): Признаки для тестирования модели
  - Масштабированы с тем же scaler, что и train
  
- **y_train.csv** (1271 × 1): Целевые значения для обучения
  - Содержит оценки качества вина (quality)
  
- **y_test.csv** (318 × 1): Целевые значения для тестирования

### Метаданные
- **feature_importance.csv**: Важность каждого признака, отсортирована по убыванию
  - Колонки: `feature`, `importance`
  
- **important_features.txt**: Список выбранных признаков в порядке важности
  - Один признак на строку
  
- **scaler.pkl**: Сохраненный объект StandardScaler
  - Используется для нормализации новых данных перед предсказанием
  - Расширение: pickle (binary format)

## Использование

### Запуск обработки данных

```bash
python scripts/data_preprocessing.py
```

### С параметрами

```bash
# Пользовательская доля теста
python scripts/data_preprocessing.py --test-size 0.3

# С новым random state
python scripts/data_preprocessing.py --random-state 123

# Переустановить порог важности признаков
python scripts/data_preprocessing.py --feature-threshold 0.02

# Переиспользовать сырой датасет с UCI
python scripts/data_preprocessing.py --force-download
```

## Примечания

1. **Воспроизводимость**: Для воспроизводимых результатов используется `random_state=42`

2. **Стратификация**: Используется стратификация по целевой переменной для сохранения распределения качества в train/test выборках

3. **Масштабирование**: Критично для алгоритмов, чувствительных к масштабу (например, SVM, KNN, градиентные методы)

4. **Scaler**: Сохраненный scaler необходимо использовать для предсказания на новых данных с помощью `scaler.inverse_transform()` для получения значений в исходной шкале
