# Лабораторная работа №4: Управление версиями данных с помощью DVC

## Цель работы
Освоить базовые операции утилиты DVC для версионирования данных, настройку удалённого хранилища и переключение между версиями датасетов.

## Ход выполнения

### 1. Настройка окружения

**1.1. Создание структуры проекта**
- Создана папка `lab4` в корне репозитория `mlops_labs`
- Инициализирован Git-репозиторий
- Инициализирован DVC: `dvc init`

**1.2. Настройка удалённого хранилища**
В качестве удалённого хранилища выбран Google Drive

### 2. Создание и версионирование датасета
***2.1. Версия 1: сырые данные***

- Загружен датасет Titanic из catboost.titanic()
- Датасет сохранён в data/titanic.csv
- Данные отправлены в удалённое хранилище

***2.2. Версия 2: заполнение пропусков в Age***

- Пропущенные значения в колонке Age заполнены средним значением
- Датасет сохранён в data/titanic_age_filled.csv
- Данные отправлены в удалённое хранилище


***2.3. Версия 3: one-hot encoding для Sex***

- Выполнено one-hot-кодирование признака Sex

- Созданы колонки Sex_female, Sex_male

- Датасет сохранён в data/titanic_ohe.csv

### 3. Результаты версионирования
***3.1. Список созданных тегов***

- v1
- v2-
- v3_ohe

### 4. Переключение между версиями
***4.1. Переключение на версию 1 (raw data)***

```bash
git checkout v1
dvc checkout
```

***4.2. Переключение на версию 2***

```bash
git checkout v2-
dvc checkout
```

***4.3. Переключение на версию 3***

```bash
git checkout v3_ohe
dvc checkout

```
***4.4. Возврат к актуальной версии***

```bash
git checkout main
dvc checkout
```

### 5. Удалённое хранилище

Мой диск/dvcstore/files/md5/
